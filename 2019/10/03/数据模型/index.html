数据模型

Posted on 2019-10-03 | Post modified: 2019-10-03 | In 数据库设计与开发 |

笔记：ヾ暖色调 「我曾一心向往天上的明月 怕今宵是庄生的蝴蝶」

E-R概念模型

  • 信息
    • 客观世界中存在的事物在人们头脑中的反映
  • 概念模型
    • 现实世界到信息世界的抽象表示
  • 实体-联系模型(Entity-Relationship data model, E-R模型)
    • 最常用的概念模型
  • 实体
    • 现实世界存在的个体，具有同类特征的个体组成的集合叫做实体集
  • 属性
    • 实体具有的若干特性，称为属性
  • 实体间的联系
    • 两个实体集间的三类联系
      • 一对一联系
      • 一对多联系
      • 多对多联系
    • 多元联系：多个实体集之间的联系
      • 三个实体间多对多联系
    • 一个实体集内部的不同实体间的联系
      • 一对一联系
      • 一对多联系
      • 多对多联系

E-R数据模型

  • E-R模型
    • 用E-R图表示现实世界中实体与实体间的联系的模型
    • 学生选课E-R图
  • 三个基本成分
    • 实体
      • 实体型用矩形表示，框中写上实体名
    • 属性
      • 描述实体的属性用椭圆框表示，用属性名标注，并将椭圆用无向边与矩形框连接
    • 联系
      • 实体之间的联系用菱形框表示，框内标明联系名，并用连线将关联的实体连接，在连线旁可以表明实体间联系的类型
      • 比如可以进一步给出实体参与联系的最小和最大次数，称为实体参与度
  • 其他概念
    • 弱实体
      • 依赖于其他实体而存在的实体称为弱实体。相对于弱实体，它所依赖的叫做强实体
      • 在E-R图中用双矩形表示弱实体，用双连线表示全部参与
    • 在扩展E-R数据模型中，增加了自类和超类的概念
      • 层次结构表示，下层实体用双竖线为边的矩形框表示，上层实体与下层实体之间用加小圈的直线连接起来，下层实体称为上层实体的子类，上层则称为超类
  • E-R模型被广泛应用于数据库概念模型的设计，在E-R图中仅表示现实世界中的信息结构和信息间的联系，不涉及信息在计算机中的存储
    • 因此只要用户需求不便，E-R模型就是稳定的，而应用E-R模型，可以很方便的转化为DBMS所支持的数据模型

层次数据模型

  • 概念
    • 层次数据模型是按照层次结构来组织数据的
      • 如图：
    • 层次结构也叫做树形结构，数中的每个节点代表一种记录类型。同时他们应该满足
      • 有且仅有根节点没有双亲
      • 其他节点有且仅有一个双亲节点
    • 层次模型中的基本数据结构是记录和由记录组成的层次结构
    • 记录是层次模型中存取数据的基本单位，由若干个字段组成
    • 优点
      • 能够很好的模拟数据间的自然层次关系，存取效率高
    • 缺点
      • 只能表示实体间1:m联系，对于非层次型的m:n联系需要引入虚拟指针或者冗余数据，增大系统开销，降低性能
    • 一个实例

网状数据模型

  • 网状数据模型允许
    • 一个节点可以多个双亲
    • 多个节点可以无双亲
  • 优点
    • 能够很好的模拟事物间各种复杂的联系
    • 用指针实现数据间的联系，存取效率高
  • 缺点
    • 复杂的导航机制以及数据定义和数据操纵语言，不容易掌握

关系数据模型

  • 关系模型的基本概念和结构
    • 关系(relation)
      • 是数据集合论的一个概念，关系模型以关系为基础发展
      • 涉及概念：关系，属性，域，元组，键
    • 关系模式(relation schema)
      • 对关系结构的描述称为关系模式(表头，表格的描述)
      • 关系名(属性名1,属性名2,...,属性名n)
      • 学生(学号,姓名,出生年月,性别,入学年份,班级)
    • 关系模式详解
      • 关系
        • 关系中的基本概念
          • 关系(relation)
            • 由行和列组成的二维表，一个关系可以描述一个实体集
          • 属性(attribute)
            • 关系中的列为这个关系中的属性，属性用属性名标识，描述实体的不同特征
          • 域(domain)
            • 一个属性对应一个值的集合，所以域就是这个集合的取值范围
          • 元组(tuple)
            • 关系是元组的集合(简单说就是元组就是关系表中的行)，一个元组对应实体集中的一个个体。元组由若干分量构成，一个分量对应一个属性
          • 键(key)
            • 键是由「一个或多个」属性组成的具有「唯一标识」元组功能的存在
            • 一个关系中可能有多组属性可以起到「唯一标识」作用，因此一个关系可以由多个键，选择其中一个为主键，其余为候选键
          • 术语对比
      • 关系模式
        • 关系模式的数据完整性约束
          • 对实体完整性约束
            • 对关系的键的约束
          • 对实体间联系的约束
            • 实体间公共域上属性值的约束
          • 用户定义的完整性约束
        • 关系模型的数据操纵
          • 关系模型中，对关系中的数据可以进行查询插入删除和修改
          • 关系运算
            • 关系代数
            • 关系演算
              • 元组关系演算
              • 域关系演算
            • 用户对数据的操纵是透明的、非过程化的，用户只需要指出做什么，不需要告诉怎么做
    • 关系模型与网状和层次模型的比较
      • 优点
        • 数据结构简单
        • 一体化的数据子语言
        • 数据独立性高
        • 面向集合的存取方式
        • 坚实的理论基础
        • 有利于开展其他应用
      • 缺点
        • 语义比较贫乏
          • 不能明确的表示实体间的联系(即不能将实体间的自然联系呈现给用户)
          • 不能很好的描述复杂对象

面向对象的数据模型(OO模型)

  • 概念
    • 二十世纪八十年代提出的，许多概念源于面向对象程序设计语言，是数据库技术和面向对象程序设计方法结合产生的
    • 核心概念
      • 对象(object)
        • 现实世界存在的一切事物都可以称为对象
      • 类(class)
        • 定义具有某些相似特征对象的对象类，代表所标识对象的类型。
        • 类的对象称为实例
        • 类中含有变量和方法
        • 由子类和超类
      • 消息(message)
        • 消息是发送给某个对象的动作标识符，它指示对象执行某些动作
        • 由于对象是封装的，对对象中的实例变量和方法的存取和调用只能通过消息的收发来实现
      • 方法(method)
        • 方法是消息的实现，是一组操作
      • 封装(encapsulation)
        • 封装是一种信息隐蔽技术，把对象的特征和行为隐蔽起来
      • 继承(inheritance)
        • 继承支持层次分类的观点，即继承只有在类按照层次排列时才有意义
        • 继承会继承类的特征数据方法
    • 关系模型和OO模型(面向对象数据模型)比较
    • 其他内容
      • 面向对象数据库仍然不够成熟
      • 对象-关系数据库系统是面向对象和关系模型相继诶和的产物
      • 对象-关系数据库系统兼有二者方面的特征，已得到广泛应用

小结

  • 概念模型
    • 用来表示信息结构的模型，不涉及信息在计算机中的表示
    • 常用实体-联系模型(E-R模型)
  • 数据库系统支持的模型
    • 传统的数据模型
      • 格式化数据模型
        • 层次数据模型
        • 网状数据模型
      • 关系数据模型
    • 面向对象数据模型
